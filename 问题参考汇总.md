## yt-dlp下载失败

有时下载失败可能是由于缺少cookie引起的。可以通过以下命令生成一个`cookies.txt`文件，并将其放在程序的根目录下解决问题（可在本地生成然后上传到服务器）。

> 参考链接：https://github.com/yt-dlp/yt-dlp/wiki/FAQ

```bash
yt-dlp --cookies-from-browser chrome --cookies cookies.txt
```

## Could not load library libcudnn_ops_infer.so.8

此错误通常是由于找不到库文件路径，可以通过设置`torch`的路径来解决，下面的命令可以用来设置环境变量`LD_LIBRARY_PATH`

> 参考链接：https://github.com/SYSTRAN/faster-whisper/issues/516

```bash
export LD_LIBRARY_PATH=`python3 -c 'import os; import torch; print(os.path.dirname(os.path.dirname(torch.__file__)) +"/nvidia/cudnn/lib")'`:$LD_LIBRARY_PATH
# 你也可以尝试以下命令
# export LD_LIBRARY_PATH=`python3 -c 'import os; import nvidia.cublas.lib; import nvidia.cudnn.lib; print(os.path.dirname(nvidia.cublas.lib.__file__) + ":" + os.path.dirname(nvidia.cudnn.lib.__file__))'`:$LD_LIBRARY_PATH
```

## VAD模型下载失败 (HTTP Error 301)

在使用WhisperX进行语音识别时，可能会遇到VAD（Voice Activity Detection）模型自动下载失败的问题：

```
urllib.error.HTTPError: HTTP Error 301: Moved Permanently
```

### 问题原因

Python的 `urllib.request` 在某些情况下不能正确处理HuggingFace的重定向，导致从以下地址下载模型时失败：
```
https://huggingface.co/Synthetai/whisperx-vad-segmentation/resolve/main/pytorch_model.bin
```

### 解决方案：手动下载模型

#### 1. 下载模型文件

使用支持重定向的工具下载模型，选择以下任一方法：

**方法A：使用wget（推荐）**
```bash
wget -O whisperx-vad-segmentation.bin https://huggingface.co/Synthetai/whisperx-vad-segmentation/resolve/main/pytorch_model.bin
```

**方法B：使用curl**
```bash
curl -L -o whisperx-vad-segmentation.bin https://huggingface.co/Synthetai/whisperx-vad-segmentation/resolve/main/pytorch_model.bin
```

**方法C：浏览器下载**
直接在浏览器中打开以下链接下载：
```
https://huggingface.co/Synthetai/whisperx-vad-segmentation/resolve/main/pytorch_model.bin
```

#### 2. 确定目标路径

运行以下命令查看模型应该存放的位置：

```bash
python3 -c "import torch; import os; print(os.path.join(torch.hub._get_torch_home(), 'whisperx-vad-segmentation.bin'))"
```

常见路径：
- **Linux/Docker**: `~/.cache/torch/hub/whisperx-vad-segmentation.bin`
- **macOS**: `~/.cache/torch/hub/whisperx-vad-segmentation.bin`
- **Windows**: `C:\Users\<用户名>\.cache\torch\hub\whisperx-vad-segmentation.bin`

#### 3. 放置模型文件

将下载的文件重命名并移动到目标路径：

```bash
# 创建目录（如果不存在）
mkdir -p ~/.cache/torch/hub

# 移动文件
mv whisperx-vad-segmentation.bin ~/.cache/torch/hub/
```

#### 4. 验证文件

检查文件大小（应该约为 17-18 MB）：

```bash
ls -lh ~/.cache/torch/hub/whisperx-vad-segmentation.bin
```

**可选：验证SHA256校验和**
```bash
sha256sum ~/.cache/torch/hub/whisperx-vad-segmentation.bin
```

期望输出：`0b5b3216d60a2d32fc086b47ea8c67589aaeb26b7e07fcbe620d6d0b83e209ea`

> 注意：如果已注释掉代码中的SHA256校验，则无需进行此验证。

### 一键下载脚本

在远程环境或服务器上执行以下完整脚本：

```bash
# 查看目标路径
TARGET_PATH=$(python3 -c "import torch; import os; print(os.path.join(torch.hub._get_torch_home(), 'whisperx-vad-segmentation.bin'))")
echo "目标路径: $TARGET_PATH"

# 创建目录
mkdir -p $(dirname $TARGET_PATH)

# 下载模型
wget -O $TARGET_PATH https://huggingface.co/Synthetai/whisperx-vad-segmentation/resolve/main/pytorch_model.bin

# 验证文件大小
ls -lh $TARGET_PATH

echo "✅ VAD模型下载完成"
```

### 代码修复方案（可选）

如果想修改代码以自动处理重定向，可以编辑 `/opt/conda/envs/linly_dubbing/lib/python3.10/site-packages/whisperx/vad.py` 文件，在第29行附近修改：

```python
# 原代码
with urllib.request.urlopen(VAD_SEGMENTATION_URL) as source, open(model_fp, "wb") as output:

req = urllib.request.Request(VAD_SEGMENTATION_URL, headers={'User-Agent': 'Mozilla/5.0'})
with urllib.request.urlopen(req) as source, open(model_fp, "wb") as output:
```
